Of course. This is the classic Tower of Hanoi puzzle. The most efficient way to solve it is using a recursive algorithm.

### The Strategy

The core idea is to break the problem down into smaller, identical problems. To move a stack of $n$ disks from a source peg to a target peg, we follow these three steps:

1.  Move the top $n-1$ disks from the **source** peg to the **auxiliary** peg.
2.  Move the single largest disk (disk $n$) from the **source** peg to the **target** peg.
3.  Move the $n-1$ disks from the **auxiliary** peg to the **target** peg.

The "base case" for this recursion is when we need to move only one disk ($n=1$), which is a single, direct move.

### Exploration with a Smaller Example (n=3)

Let's apply this strategy to the 3-disk example you provided, moving the stack from peg 0 to peg 2, using peg 1 as the auxiliary.

**Goal**: Move 3 disks from peg 0 to peg 2.

1.  **Move 2 disks from peg 0 to peg 1 (auxiliary).**
    1.  **Move 1 disk from peg 0 to peg 2 (new auxiliary).**
        - Move disk 1 from 0 to 2. `[1, 0, 2]`
    2.  **Move disk 2 from peg 0 to peg 1.**
        - Move disk 2 from 0 to 1. `[2, 0, 1]`
    3.  **Move 1 disk from peg 2 to peg 1.**
        - Move disk 1 from 2 to 1. `[1, 2, 1]`
2.  **Move disk 3 from peg 0 to peg 2.**
    - Move disk 3 from 0 to 2. `[3, 0, 2]`
3.  **Move 2 disks from peg 1 to peg 2.**
    1.  **Move 1 disk from peg 1 to peg 0 (new auxiliary).**
        - Move disk 1 from 1 to 0. `[1, 1, 0]`
    2.  **Move disk 2 from peg 1 to peg 2.**
        - Move disk 2 from 1 to 2. `[2, 1, 2]`
    3.  **Move 1 disk from peg 0 to peg 2.**
